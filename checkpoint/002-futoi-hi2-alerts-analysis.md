# 002 — FUTOI + HI2 + Alerts: загрузка и корреляционный анализ

**Дата:** 2026-06-21
**Статус:** все датасеты загружены, correlation scan проведён

## Что изменилось

### Новые датасеты

Добавлены три новых источника данных поверх существующих `tradestats_fo`/`obstats_fo`/`orderstats_fo`:

1. **FUTOI** (MOEX ISS API) → `moex.futoi`
   - Строк: **1,578,249** (78 тикеров, 2020-01-03 — 2026-06-12)
   - Данные: внутридневные снапшоты позиций FIZ/YUR по каждому фьючерсу
   - Поля: `sess_id, seqnum, ticker, clgroup, pos, pos_long, pos_short, pos_long_num, pos_short_num, trade_session_date`

2. **HI2** (AlgoPack) → `moex.hi2_fo`
   - Строк: **1,138,365** (2020-01-03 — 2026-06-12)
   - Метрики: `hhi_volume, hhi_agressive, hhi_passive, hhi_buy/sell, hhi_netflow` — **Herfindahl-Hirschman Index** концентрации рынка

3. **Alerts** (AlgoPack) → `moex.alerts_fo`
   - Строк: **331,322** (2024-01-03 — 2026-06-12)
   - Типы: `pr_low_min`, `vol_99_9_pctl`, `oi_close_change_99_9_pctl`, `net_vol_99_9_pctl+/-` — события 99.9 перцентиля

### Исправления

- **FUTOI конвертер:** исправлена ошибка `NoneType` в `trade_session_date` — добавлен `_conv_date_or_none()`, который не передаёт None в clickhouse_connect для `Date` колонки
- **Создана таблица `futoi` на 10.0.0.60** (не создавалась автоматически)
- **Скопированы `hi2_fo` и `alerts_fo`** с 10.0.0.63 на 10.0.0.60 (загрузчик пишет на 63, потребители читают с 60)
